# 🌐 Domain Investigations

> Analysing domains, DNS records, and internet infrastructure through open-source intelligence and evidence-based investigation — from a single domain to a fully mapped network of associated infrastructure.

Every investigation in this folder starts with a target domain or suspicious indicator and ends with a documented, reproducible intelligence assessment. No assumptions. No shortcuts. Evidence in, intelligence out.

---

## 🎯 What This Covers

| Technique | What It Involves |
|---|---|
| **Domain Profiling** | Building a comprehensive intelligence picture of a domain's history, ownership, and activity |
| **DNS Analysis** | Examining DNS records, name servers, and resolution patterns to map infrastructure |
| **WHOIS Research** | Extracting and analysing domain registration data to identify ownership indicators |
| **Infrastructure Mapping** | Charting hosting providers, IP ranges, and associated network infrastructure |
| **Subdomain Enumeration** | Discovering and analysing subdomains to expand the infrastructure picture |
| **Historical Analysis** | Reviewing historical DNS records, WHOIS data, and archived content for investigative leads |
| **Reputation Assessment** | Evaluating domain reputation, threat history, and abuse indicators across open sources |
| **Threat Detection** | Identifying domains associated with malicious activity, phishing, or infrastructure abuse |
| **Attribution Analysis** | Linking domains to individuals, organisations, or threat actors using available evidence |
| **Evidence Documentation** | Recording all findings in structured, verifiable, and reproducible investigation reports |

## 🔬 Investigative Methodology

Each investigation follows a disciplined, repeatable eight-stage workflow:

```
╔══════════════════════════════════════════════════════════════╗
║               INVESTIGATIVE METHODOLOGY                      ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  01  RESEARCH QUESTION                                       ║
║      Define the target domain and intelligence requirement   ║
║                              ↓                               ║
║  02  COLLECTION                                              ║
║      Gather domain records, DNS data, WHOIS history,         ║
║      and publicly available infrastructure intelligence      ║
║                              ↓                               ║
║  03  VERIFICATION                                            ║
║      Validate source credibility and data integrity          ║
║                              ↓                               ║
║  04  CORRELATION                                             ║
║      Cross-reference findings across multiple platforms      ║
║      and independent intelligence sources                    ║
║                              ↓                               ║
║  05  ANALYSIS                                                ║
║      Apply domain intelligence techniques to map             ║
║      infrastructure, ownership, and threat indicators        ║
║                              ↓                               ║
║  06  FINDINGS                                                ║
║      Establish key conclusions on domain legitimacy,         ║
║      ownership, and associated risk                          ║
║                              ↓                               ║
║  07  INTELLIGENCE ASSESSMENT                                 ║
║      Assess confidence level, attribution confidence,        ║
║      and alternative hypotheses                              ║
║                              ↓                               ║
║  08  REPORT                                                  ║
║      Document conclusions in a structured, reproducible      ║
║      domain intelligence report                              ║
║                                                              ║
╚══════════════════════════════════════════════════════════════╝
```

Every conclusion is evidence-backed. Every step is reproducible.

## ⚠️ Responsible Use

All investigations in this folder are conducted strictly using open-source intelligence and publicly available data:

- **Legal compliance** — All methods comply with applicable laws and platform terms of service
- **Public data only** — No unauthorised access, active scanning, or exploitation of live infrastructure
- **Responsible handling** — Sensitive findings and attribution conclusions are handled with appropriate care
- **Educational purpose** — All case studies are conducted for learning and portfolio demonstration only
